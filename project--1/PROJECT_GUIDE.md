# Hungry Ninja — 项目总览

> 餐馆建站展示项目 | 拿 Demo 谈客户 | 有需求再定制

---

## 核心逻辑

拿 Hungry Ninja 做 Demo → 给潜在餐馆老板看 → 他们有需求再跟进定制

---

## 当前决策一览

| 项目 | 决策 |
|------|------|
| **Demo 客户** | Hungry Ninja（Stratford 真实餐馆，仅做展示用） |
| **目标客户** | Google Maps 有评价但没网站的国外餐馆老板 |
| **语言** | 多语言按客户需求配置（Demo：中/英/日） |
| **前端** | Replit 主力（UI 做得好 + 自带框架依赖） |
| **后端预约** | 暂不开发，客户有需求再定方案 |
| **私库方案** | 暂不建，等有真实客户再说 |
| **部署** | 暂不上线，本地 + Replit 够展示用 |
| **设计指南** | Replit 端 DESIGN_GUIDE.md 为权威来源 |
| **图片处理** | Replit 主力，mcp-openrouter 备用 |

## 各 Phase 详细内容

### Phase 1 — 选店 + 素材收集 ✅

| 内容 | 说明 |
|------|------|
| **做的事** | 在 Google Maps 上找合适的餐馆，截图收集素材 |
| **输出** | `attached_assets/` 里的所有图片 |
| **选店逻辑** | Google Maps 有评价积累、没有网站、风格适合展示 |

---

### Phase 2 — 设计对齐 ✅

| 内容 | 说明 |
|------|------|
| **做的事** | 分析餐厅实景图片，提炼色彩/字体/风格 → 出设计指南 |
| **输出** | `DESIGN_GUIDE.md` |
| **角色** | 我主导，Replit 辅助看图 |

---

### Phase 3 — 基础框架搭建 ✅

| 内容 | 说明 |
|------|------|
| **做的事** | React + Vite + Tailwind 项目初始化，首页 + 导航 + 响应式 |
| **输出** | `artifacts/hungry-ninja/` 全套前端项目 |
| **角色** | Replit 按设计指南执行，我本地验收 |
| **技术栈** | React 19 / Vite 7 / Tailwind v4 / shadcn-ui / Framer Motion |

---

### Phase 4 — 菜单展示模块 ✅

- 分类 Tab（All / Sushi / Bowls / Salads）
- 菜品卡片（图片 + 名称 + 价格 + 描述）
- 筛选切换动画（Framer Motion）
- 图片预加载 + 加载失败重试

---

### Phase 5 — 顾客评价模块 ✅

- 4.6★ 星级评分
- 6 条真实 Google 评价引用
- View on Google Maps 链接

---

### Phase 6 — 预约 + 后台（暂缓）

| 内容 | 说明 |
|------|------|
| **计划方案** | 表单提交 → API 存 DB → 老板登录查看 → 标记已处理 |
| **技术栈** | Express + PostgreSQL (Drizzle ORM) |
| **安全** | CORS / 速率限制 / JWT 认证 |
| **状态** | 客户有需求再启动 |

---

### Phase 7 — 风格细化 + SEO + 多语言（当前重点）

#### 多语言

| 语言 | 状态 |
|------|------|
| 英文 (en.ts) | ✅ 已实现 |
| 中文 (zh.ts) | ✅ 已实现 |
| 日文 (ja.ts) | ✅ 已实现 |

#### SEO

| 项目 | 说明 |
|------|------|
| Schema.org 结构化标记 | 添加 LocalBusiness / Restaurant 标记，让 Google 搜索结果直接显示评分、营业时间、地址 |
| 本地 SEO | NAP（名称/地址/电话）一致性 + Google Maps 优化 |
| 速度优化 | 图片懒加载、代码分割、缓存策略 |
| Meta 完善 | 每个页面独立 title/description，OG 标签增强 |
| Sitemap | 自动生成 sitemap.xml + robots.txt 完善 |

#### 风格细化

- 微调视觉细节
- 动效打磨
- 内容文案最终确认
